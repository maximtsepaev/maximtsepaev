# Портфолио разработки / Go Backend Developer

Репозиторий содержит базу моих проектов, практических кейсов и архитектурных решений в области бэкенд-разработки на Go. Основной фокус: проектирование отказоустойчивых сервисов, микросервисная архитектура, многопоточность и оптимизация работы с реляционными базами данных.

---

## Технический стек и экспертиза

* **Разработка на Go**: Глубокое понимание рантайма, управление конкурентностью (Goroutines, Channels, Mutex, WaitGroup, Context), локализация и устранение Race Conditions, профилирование памяти и безопасная работа с общими ресурсами[cite: 1].
* **Архитектура**: Проектирование систем по принципам Clean Architecture (слои Domain, UseCase, Repository), разработка строго типизированных контрактов взаимодействия, построение REST API и распределенных микросервисных систем[cite: 1].
* **Работа с данными**: Проектирование реляционных схем в PostgreSQL, управление пулами соединений через database/sql и pgx, оптимизация сложных транзакций, реализация бизнес-логики на уровне строгих SQL constraints[cite: 1].
* **Сетевой слой и безопасность**: Кастомные цепочки Middleware (логирование, сбор метрик, перехват паник), JWT-аутентификация, валидация входящих потоков данных, кастомизация HTTP и gRPC серверов[cite: 1].
* **Инфраструктура**: Контейнеризация приложений (Docker, Docker Compose), автоматизация тестирования и сборки через CI/CD (GitHub Actions), администрирование Linux-сред (Bash)[cite: 1].

## Ключевые проекты в репозитории

### Go Final Project (Флагманский бэкенд-сервис)
Центральный проект, демонстрирующий построение масштабируемого серверного приложения с нуля по стандартам чистой архитектуры.

* **Архитектурное решение**: Полное разделение приложения на независимые слои. Бизнес-логика (UseCases) изолирована от деталей реализации (HTTP-роутеры, драйверы БД), что позволяет легко тестировать и модифицировать компоненты системы.
* **Реализованный функционал**:
  * Разработан производительный REST API с централизованной валидацией и парсингом входящих JSON-структур[cite: 1].
  * Интегрирован слой безопасности: авторизация на базе JWT-токенов и разграничение прав доступа через Middleware[cite: 1].
  * Реализован отказоустойчивый слой работы с PostgreSQL: безопасная обработка транзакций, предотвращение утечек пула соединений, оптимизация структуры таблиц для быстрых выборок[cite: 1].
  * Проект полностью контейнеризирован (Docker Compose) и снабжен механизмами логирования и конфигурации через переменные окружения[cite: 1].

### Flashsale (Проектирование высоконагруженного микросервисного бэкенда)
*Проект в активной фазе разработки / Архитектурная заглушка*

Распределенная система, спроектированная специально для обработки экстремальных пиковых нагрузок в моменты краткосрочных распродаж и акций.

* **Архитектура системы**: Монолитная логика разбита на изолированные доменные микросервисы — Inventory Service (контроль остатков) и Order Service (оформление заказов).
* **Инженерные вызовы и решения**:
  * **Межсервисное взаимодействие**: Реализован переход с HTTP на gRPC/Protobuf. Самостоятельно описаны `.proto` контракты, что снизило сетевой оверхед и обеспечило строгую типизацию обмена данными.
  * **Консистентность и Race Conditions**: Разработана схема базы данных в PostgreSQL с использованием жестких SQL constraints на уровне СУБД. Это гарантирует полную блокировку покупки при нулевом остатке на складе, исключая race condition при обработке тысяч конкурентных запросов в секунду.
  * **Конкурентная обработка**: Написание внутренней логики сервисов с применением примитивов синхронизации Go для эффективного распределения нагрузки на CPU.

## Технологический лог: Этапы разработки и практика

В профиле представлены репозитории, отражающие последовательное освоение и практическую отработку ключевых концепций Go-платформы (в общей сложности реализовано более 10 бэкенд-компонентов)[cite: 1]:

* **Многопоточность и асинхронность**: Написание конкурентных кэшей, реализация паттерна Worker Pool для параллельной обработки задач, управление жизненным циклом потоков и таймаутами через пакет Context[cite: 1].
* **Интеграция сетевых протоколов**: Написание HTTP-клиентов и серверов, парсинг данных, разработка кастомных роутеров иMiddleware для сквозного логирования системных событий[cite: 1].
* **Слой хранения и СУБД**: Практика написания чистых SQL-запросов, организация миграций базы данных, работа со сложными выборками, индексами и внешними ключами[cite: 1].
* **DevOps-конвейер**: Автоматизация тестирования кода и контроля качества (микро-тесты, линтеры) при каждом коммите с помощью GitHub Actions[cite: 1].
