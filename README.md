## Технологический стек

* **Язык**: Go (Golang)
* **Конкурентность**: Goroutines, channels, mutex, waitgroup, context, предотвращение race condition и безопасная работа с общими данными.
* **Сеть и API**: HTTP, REST API, gRPC, JWT-аутентификация, написание middleware, логирование.
* **Базы данных**: PostgreSQL, проектирование реляционных БД, написание SQL-запросов и ограничений (SQL constraints).
* **Окружение и Инфраструктура**: Linux, написание Bash-скриптов, Docker, Docker-compose, Git (GitHub), основы CI/CD (GitHub Actions).

## Проекты

### [Go Final Project](https://github.com/maximtsepaev/go_final_project)
Готовое серверное бэкенд-приложение, написанное по итогам курса обучения.
* **Что внутри**:
  * Реализовано REST API для обработки HTTP-запросов.
  * Подключена база данных PostgreSQL, написаны SQL-запросы для сохранения и выборки данных.
  * Настроена JWT-аутентификация для пользователей и middleware для проверки прав.
  * Приложение упаковано в Docker и запускается через Docker-compose.

### [Flashsale](https://github.com/maximtsepaev/flashsale) *(В разработке)*
* **Идея**: Разработка бэкенда для системы мгновенных распродаж, способной обрабатывать одновременные запросы от множества пользователей.
* **Что реализуется**:
  * Разделение логики на сервисы для заказов и склада (микросервисный подход).
  * Использование gRPC для быстрого и типизированного взаимодействия между компонентами.
  * Настройка безопасной работы со складскими остатками в PostgreSQL (SQL constraints), чтобы товар не уходил в минус при конкурентных запросах.
  * Защита от race condition при одновременной покупке одного и того же товара.
